# DocStation Linux v8

Linux-ветка DocStation для Ubuntu. В v3 web-интерфейс больше не основной: добавлено нормальное нативное GTK-приложение под тач-экран.

## Быстрый старт

```bash
cd ~/Downloads/docstation_linux
chmod +x scripts/*.sh
./scripts/install_ubuntu.sh
sudo reboot
```

После перезагрузки:

```bash
cd ~/Downloads/docstation_linux
./scripts/run_app.sh
```

Также установщик создаёт ярлык `DocStation` на рабочем столе.

Пароли по умолчанию:

```text
оператор: 111
админ: 888
```

## Что есть в нативном приложении

- крупный тач-вход с цифровой клавиатурой;
- роли оператор/админ;
- главный экран со статусами: всего / очередь скачивания / подключены / скачивание / готовы / ошибки;
- запуск полного цикла импорта;
- сухая проверка регистраторов без перевода в диск;
- архив с поиском по всем словам и всем полям;
- просмотр файла через VLC/системный просмотрщик;
- копирование файла оператором и администратором;
- защита/снятие защиты и удаление только для администратора;
- журнал действий для администратора;
- просмотр карты стаканов;
- запуск калибровки стаканов в терминале;
- смена пароля текущей роли через тач-клавиатуру.

## Калибровка стаканов

```bash
./scripts/calibrate_slots.sh 20
```

Проверка:

```bash
./scripts/show_slots.sh
./scripts/list_amba.sh
```

Карта хранится в `DockSlots.json`.

## Один полный цикл импорта из консоли

```bash
./scripts/run_once.sh
```

## Настройки

`StationConfig.linux.json`:

```json
{
  "storage_root": "archive",
  "database_path": "data/docstation.sqlite3",
  "dock_slots_path": "DockSlots.json",
  "switch_to_disk_delay_ms": 10000,
  "max_parallel_imports": 6,
  "max_parallel_imports_per_hub": 3
}
```

Для реальной док-станции архив лучше вынести на отдельный диск, например:

```json
"storage_root": "/mnt/docstation_archive",
"database_path": "/var/lib/docstation/docstation.sqlite3"
```

## v7: автоматическая калибровка стаканов

`./scripts/calibrate_slots.sh 20` теперь работает без Enter на каждом стакане: вставил регистратор — путь зафиксировался — вынул — программа сама перешла к следующей ячейке.

## v8: полноэкранное приложение и ячейки

- Главное приложение запускается полноэкранно.
- На главном экране теперь видны ячейки док-станции.
- Папка сохранения архива задаётся в приложении: `Настройки → Папка архива`.
- Задержка перевода регистратора в диск по умолчанию: `5000 мс`.
- По умолчанию архив: `/mnt/docstation_archive`.

Оконный режим для отладки:

```bash
DOCSTATION_WINDOWED=1 ./scripts/run_app.sh
```


## v9

Карточки ячеек на главном экране доведены ближе к Windows-версии: камера, сотрудник, зарядка-placeholder, память, прогресс скачивания, текущий файл, скорость и ETA.

## v11: автоматический операторский режим

В нативном приложении больше не нужно запускать цикл скачивания вручную. После входа приложение само держит службу активной: найденный регистратор сразу считывается, ставится в очередь скачивания, переводится в диск и скачивается при наличии свободного параллельного слота.

Карточки ячеек рассчитаны на полноэкранный режим 1920×1080 и не должны менять размер от текста внутри.


## v13: режим ожидания в Ambarella

Нативное приложение больше не переводит все обнаруженные регистраторы в Mass Storage заранее. Регистратор остаётся в Ambarella/vendor mode после чтения ID/сотрудника/памяти и переводится в диск только тогда, когда есть свободный слот скачивания на его USB-ветке.
