# DocStation Linux v13 — Mass Storage только на время скачивания

Изменена логика постоянного рабочего цикла нативного приложения.

## Было

1. Обнаружили регистратор в Ambarella/vendor mode.
2. Считали информацию.
3. Последовательно переводили регистраторы в Mass Storage.
4. Потом скачивали с ограничением по потокам.

При большом количестве регистраторов получалось много устройств в режиме Mass Storage, которые просто ждут очереди. Это сильнее нагружает USB/storage stack и может снижать стабильность.

## Стало

1. Обнаружили регистратор в Ambarella/vendor mode.
2. Сразу считали Device ID, Officer ID и память.
3. Показали информацию в ячейке.
4. Оставили регистратор в Ambarella/vendor mode.
5. Ждём свободного слота скачивания на его USB-ветке/хабе.
6. Только перед реальным скачиванием переводим конкретный регистратор в Mass Storage.
7. Диск появился — сразу стартует импорт.
8. После импорта диск размонтируется, ячейка получает статус «готово».

## Лимиты

Параметры сохранились:

```json
"max_parallel_imports": 6,
"max_parallel_imports_per_hub": 3
```

Один хаб не забирает лимит другого. Если на одной USB-ветке работа закончилась, на второй всё равно максимум 3 активных скачивания.

- подключение к архиву;
- подключён;
- скачивание;
- готово;
