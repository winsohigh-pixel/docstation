# DocStation Linux v12 — распределение скачиваний по USB-веткам

Добавлен hub-aware планировщик импорта.

## Логика

- Общий лимит скачиваний: `max_parallel_imports`, по умолчанию `6`.
- Лимит на одну USB-ветку/хаб: `max_parallel_imports_per_hub`, по умолчанию `3`.
- USB-ветка определяется автоматически по первому компоненту `usb_path`/USB topology path. Например `3.7.4` относится к ветке `3`, а `14.7.1` — к ветке `14`.
- Если на одном хабе регистраторы закончились, второй хаб не забирает его лимит: остаётся максимум `3` потока на оставшейся ветке.

## Зачем

Для схемы `2 × 10 USB 2.0 hub` это не даёт случайно забить один хаб всеми потоками. Планировщик держит примерно `3 + 3`, а после завершения очередного регистратора запускает следующий из той же USB-ветки, если он есть.

## Конфиг

```json
{
  "max_parallel_imports": 6,
  "max_parallel_imports_per_hub": 3
}
```

Эти параметры доступны в нативном приложении в разделе `Настройки`.
