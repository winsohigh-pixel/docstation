# DocStation Linux v11 — непрерывный операторский режим

Изменения относительно v10:

- Главное окно остаётся полноэкранным, но сетка ячеек теперь фиксирована под экран 1920×1080.
- Карточки ячеек имеют фиксированный размер и не меняют высоту/ширину от длины текста.
- Технический USB-порт/USB-path больше не выводится на карточке оператора.
- При обнаружении регистратора приложение сразу читает стартовую информацию в Ambarella-режиме и заполняет карточку: регистратор, сотрудник, память.
- Память в карточке теперь показывает занятый объём полезных записей, а не свободное место.
- На главном экране больше нет кнопки «Запустить скачивание»: служба импорта запускается автоматически вместе с приложением.
- Цикл больше не ждёт, пока все регистраторы обнаружатся и переключатся в диск.
- Каждый регистратор берётся в работу сразу: обнаружен → считана информация → очередь подключения → последовательный перевод в диск → немедленный запуск скачивания при наличии свободного параллельного слота.
- Параллельное скачивание по-прежнему ограничивается параметром `max_parallel_imports`.

Операторский алгоритм:

1. Оператор вставляет регистратор.
2. Карточка стакана сразу показывает обнаружение.
3. Программа читает Device ID / Officer ID / память в Ambarella-режиме.
4. Регистратор ставится в очередь подключения.
5. Подключение в режим диска выполняется строго по одному, с паузой `switch_to_disk_delay_ms`.
6. Как только конкретный регистратор перешёл в диск, он сразу попадает в очередь скачивания.
7. Скачивание стартует немедленно, если не превышен лимит `max_parallel_imports`.
