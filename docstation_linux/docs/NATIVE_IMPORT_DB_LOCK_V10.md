# DocStation Linux v10 — исправление параллельного импорта

Исправлено падение при старте импорта нескольких дисков:

- SQLite-доступ сериализован через `threading.RLock` в `StationDatabase`.
- Увеличен `busy_timeout` до 30 секунд.
- Включён `PRAGMA synchronous=NORMAL` вместе с WAL.
- Исправлен обработчик ошибки в GTK-приложении: исключение теперь корректно передаётся в UI-поток и не вызывает `NameError`.
- Добавлена локальная проверка параллельных записей в базу.

Причина: параллельные импорт-потоки одновременно писали статусы/журнал/медиа-записи через один `sqlite3.Connection`, что давало ошибку `cannot start a transaction within a transaction`.
